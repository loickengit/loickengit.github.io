Lexical Analysis

Regular Expression

We need regular expressions to match word in lexical analysis.

Introduction of regular exp.

Def. The regular expression over epsilon are the smallest set of expressions including

Single character, Epsilon, Union, Cancatenation, Iteration

Goal of lexical analysis

Classify program substrings according to role

Communicate tokens to the parser.

  1. Partition the input string into lexemes. This is implemented by reading left-to-right, recognizing one token at a time.

  2. “Lookahead” may be required to decide where one token ends and the next token begins.

Target tokens

  • Identifier:

    strings of letters or digits, starting with a letter

  • Integer

    a non-empty string of digits

  • Keyword

    “else” or “if” or “begin” or….

  • Whitespace

    a not-empty sequence of blanks, newlines, and tabs

How to describe Integer, String… in regular exp ?

Integer: a non-empty string of digits

digit = ‘0’ + ‘1’, ‘2’ +….’9’

digit digit* => digit+

Identifier: strings of letters or digits, starting with a letter

letter = [a-zA-Z]

letter(letter+digit)*

Whitespace: a non-empty sequence of blanks, newlines, and tabs

(‘ ‘ + ‘\n’, ‘\t’)+

A specification for the predicate

  1. Write a rexp for the lexemes of each token class

    1. Number= digit+
    2. Keyword= “if”, “else”….
    3. Identifier = letter(letter+digit)*
    4. OpenPar = “(“
    5. ……

  2. Construct R, mathing all lexemes for all tokens

    R = Keyword + Identifier + Number +…..

  3. Let input be x1……xn

    for 1 <= i <= n check x1…xi <- L(R)

  4. If success, then we know that

    x1…xi <- L(Rj) for some j

  5. Remove x1…xifrom input and go to (3)
